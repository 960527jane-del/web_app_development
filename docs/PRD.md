# 專案需求文件 (PRD) - 二手書交換與買賣平台

## 1. 專案概述

### 背景與動機
大學課程經常需要購買教科書，而這些書籍在學期結束後往往閒置。為了有效利用資源、減少學生的經濟負擔，並提倡環保永續，我們希望建立一個專為大學生設計的二手書交換與買賣平台。

### 目標用戶
- 主要目標用戶為**大學生**。

### 核心價值主張
- **安全單純**：限定大學生註冊，減少一般外部人士造成的詐騙與糾紛。
- **快速媒合**：依據書名、課程或科系分類，讓學生快速找到所需教材。
- **減少浪費**：促進校園內二手書籍的循環利用，讓知識持續傳承並降低購書成本。

## 2. 功能需求

以下列出平台的核心功能，並以使用者故事（User Story）格式描述：

### 功能一：學生帳號註冊與驗證
- **作為** 一名大學生，
- **我希望** 能夠註冊平台帳號並驗證學生身分，
- **以便** 確保平台上的交易對象都是真實的學生，增加交易的安全與可信度。

### 功能二：線上書櫃與書籍上架
- **作為** 一名賣家或想交換書籍的人，
- **我希望** 能夠輕鬆上傳二手書的資訊（包含書名、課程名稱、書況、照片與價格），
- **以便** 將我閒置的書籍展示在平台上讓其他同學選購。

### 功能三：商品預覽與庫存查看
- **作為** 一名潛在買家，
- **我希望** 能夠在網站上瀏覽書籍的詳細資訊，並查看目前是否還有庫存（或是否已售出），
- **以便** 決定是否要進行購買。

### 功能四：書籍搜尋與無縫探索
- **作為** 一名正在尋找特定教科書的學生，
- **我希望** 能透過關鍵字搜尋（書名、作者或課程名稱），
- **以便** 快速精準地找到我這學期需要的指定教材。

### 功能五：訊息聯絡與交易溝通
- **作為** 買賣雙方，
- **我希望** 能夠在平台上找到對方的聯絡方式或發送站內留言，
- **以便** 討論面交細節、書籍真實狀況或是進行議價。

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 搭配 Flask。
- **網頁渲染**：使用 Jinja2 模板引擎與 HTML/CSS/JavaScript，打造具備美感與現代感的介面。
- **資料庫**：使用 SQLite 作為初期資料庫儲存方案。

### 效能與安全考量
- 確保密碼等敏感資料在資料庫中經過適當雜湊加密（如 bcrypt）。
- 實作防護機制避免未授權的修改（例如，只有書籍擁有者才能編輯或下架書籍）。
- 上傳圖片時控制檔案大小限制，提高頁面載入效能與系統穩定性。

## 4. MVP 範圍 (最小可行產品)

| 優先級 | 功能說明 |
| :--- | :--- |
| **Must Have (必須有)** | 1. 使用者註冊、登入系統<br>2. 新增、編輯、下架二手書清單<br>3. 首頁商品列表預覽與單一商品詳情頁<br>4. 關鍵字書籍搜尋功能 |
| **Should Have (應該有)** | 1. 買賣雙方留言板或聯絡資訊顯示<br>2. 簡易的個人頁面（顯示我的上架書籍） |
| **Nice to Have (加分項)** | 1. 我的收藏口袋清單<br>2. 交易狀態切換（上架中、保留中、已售出）<br>3. 使用者基本評價功能 |
